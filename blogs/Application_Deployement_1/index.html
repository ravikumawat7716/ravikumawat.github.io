Deploying a Flask Application on AWS EC2 with Gunicorn and Nginx

This guide will walk you through the process of deploying a Flask application on an AWS EC2 instance using Gunicorn as the WSGI server and Nginx as the reverse proxy.

Prerequisites

  • An AWS account.
  • Basic knowledge of command line and SSH.
  • A Flask application ready for deployment.

Step 1: Launch an EC2 Instance

  1. Sign in to your AWS Management Console.
  2. Navigate to the EC2 Dashboard.
  3. Click on "Launch Instance."
  4. Choose an Amazon Machine Image (AMI): Select an Ubuntu Server (e.g., Ubuntu 20.04 LTS).
  5. Choose an Instance Type: Select an instance type (e.g., t2.micro).
  6. Configure Security Group: Allow HTTP (port 80) and SSH (port 22).
  7. Review and Launch: Complete the setup and launch the instance.

Step 2: Connect to Your EC2 Instance

  1. Use SSH to connect to your instance: 
    ssh -i /path/to/key.pem ubuntu@your-public-ip

Step 3: Install Necessary Packages

  1. Update and upgrade: 
    sudo apt update && sudo apt upgrade -y
  2. Install Python and pip: 
    sudo apt install python3 python3-pip python3-venv -y

Step 4: Set Up Your Flask Application

  1. Clone your Flask repository: 
    git clone https://github.com/yourusername/your-repo.git
  2. Navigate and set up a virtual environment: 
    cd your-repo
python3 -m venv venv
source venv/bin/activate

Access Your Application

Navigate to your application:

http://your-public-ip
